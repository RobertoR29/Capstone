# Capstone Project: Open-Source Malware Sandbox Analysis

## Overview
This capstone project evaluated three leading open-source malware analysis platforms: CAPE Sandbox, Cuckoo3, and DRAKVUF. The objective was to determine how effectively each sandbox captures forensic artifacts during dynamic malware analysis. Our team deployed each platform in a controlled lab environment and executed 14 Windows samples spanning ransomware, worms, botnets, trojans, infostealers, rootkits, and benign software. By comparing the resulting reports, we assessed each platform's visibility into process activity, network communications, memory artifacts, file system changes, and behavioral indicators to identify the strengths and limitations of each sandbox for malware analysis.

---

## My Role
- Deployed and configured the CAPE Sandbox environment on Ubuntu Linux.
- Built and managed the virtualized malware analysis environment.
- Executed all malware and benign samples through the CAPE platform.
- Analyzed generated reports to identify behavioral indicators and malicious activity.
- Reviewed process execution, network communications, file system changes, memory artifacts, and indicators of compromise (IOCs).
- Compared CAPE results against findings from Cuckoo3 and DRAKVUF to evaluate differences in forensic visibility and malware detection capabilities.
